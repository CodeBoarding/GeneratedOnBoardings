## Input Processing Component Overview

This document provides an overview of the `Input Processing` component, which is responsible for processing input data for analysis, including parsing and locating ranges. The component consists of several interacting classes and methods, as detailed below.

### Component Descriptions

*   **Raw Input Data**: Represents the raw, unprocessed input received by the system. It is the initial data source for the analysis.
    *   **Source Files**: N/A
    *   *Interaction*: Provides the data to the `Input Object` for parsing.

*   **Input Object**: Parses the raw input data into a structured format, creating a dataflow graph and providing methods to access and interpret the data.
    *   **Source Files**: `repos.invariant.invariant.analyzer.runtime.input.Input`
    *   *Interaction*: Receives raw input, creates a `Dataflow Graph`, and uses the `Range Locator` to identify relevant ranges.

*   **Dataflow Graph**: Stores the dataflow within a given input, constructing a sequential graph per top-level list in the input.
    *   **Source Files**: `repos.invariant.invariant.analyzer.runtime.input.InputProcessor`
    *   *Interaction*: Created by the `Input Object` during parsing.

*   **Range Locator**: Locates ranges within the input using a visitor pattern, identifying and extracting relevant ranges from the input based on certain criteria.
    *   **Source Files**: `repos.invariant.invariant.analyzer.runtime.input.RangeLocator`
    *   *Interaction*: Used by the `Input Object` to locate specific ranges.

*   **Range**: Represents a range or location within the input data, pinpointing specific sections of the input that are relevant to the analysis.
    *   **Source Files**: `repos.invariant.invariant.analyzer.runtime.range.Range`
    *   *Interaction*: Located by the `Range Locator` and associated with the `Input Object`.

*   **Input Processor**: Processes the input to transform and prepare it for analysis. This might involve tokenizing, normalizing, or otherwise structuring the input data to make it easier to analyze.
    *   **Source Files**: `repos.invariant.invariant.analyzer.runtime.input.InputProcessor`
    *   *Interaction*: Receives data from the `Input Object` and prepares it for analysis.

*   **Analyzed Input**: The processed input data, ready for analysis by the policy engine.
    *   **Source Files**: N/A
    *   *Interaction*: Provides the processed data to the `Policy Engine`.

*   **Policy Engine**: Analyzes the processed input data based on defined policies.
    *   **Source Files**: N/A
    *   *Interaction*: Receives the `Analyzed Input` and applies policies to it.
